SyncForge

A backend-focused, event-driven microservices system designed to solve data consistency challenges in distributed architectures using CDC and reliable messaging patterns.

  • Domain: Distributed Systems
  • Architecture: Event-Driven Microservices
  • Tech Stack: Java, Spring Boot, Kafka, Debezium, PostgreSQL

Problem → Solution → Impact

  • Problem: Microservices experienced data inconsistency caused by the Dual Write Problem, where database commits and Kafka event publishing could fail independently.
  • Solution: Implemented the Outbox Pattern with Debezium CDC to atomically capture database changes and stream them to Kafka, combined with idempotent consumers.
  • Impact: Achieved 99.99% reliable event delivery under high load while maintaining loose coupling, fault tolerance, and scalability across services.

Architecture Highlights

  • Transactional outbox table ensured atomic database writes and event publication.
  • Debezium CDC streamed changes without requiring changes to core business logic.
  • Kafka-based asynchronous messaging enabled horizontal scalability.
  • Idempotent consumers prevented duplicate processing during retries and failures.
  • Designed for resilience, replayability, and graceful recovery from partial failures.
