Kleeflow

Kleeflow is a full-scale simulation platform for road-building and construction machines, designed to model real-world equipment behavior, production flow, and operational constraints. I led the development of the system end-to-end, spanning backend architecture, frontend PWA, and simulation logic.

  • Role: Tech Lead & Backend Architect
  • Team Size: 4 Engineers
  • Domain: Industrial Simulation Software
  • Tech Stack: Java, Spring Boot, React, PWA, REST APIs
  • Architecture: Modular Microservices + Simulation Engine

Problem → Solution → Impact

  • Problem: Existing planning tools for road-building machines lacked accurate simulation of equipment behavior, production flow, and system constraints, leading to inefficient planning and costly operational decisions.
  • Solution: Designed and built a simulation platform that models machine interactions, material flow, and operational parameters using a modular, backend-driven architecture with a real-time, interactive PWA frontend.
  • Impact: Enabled engineers and planners to simulate multiple configurations, detect bottlenecks early, and optimize machine utilization before deployment, significantly improving planning accuracy and decision-making.

System Design & Architecture

  • Designed a layered backend architecture separating simulation core logic, domain models, and REST API exposure.
  • Implemented stateless Spring Boot services to support horizontal scalability and future cloud deployment.
  • Modeled machine entities, workflows, and dependencies to accurately reflect real-world road construction processes.
  • Used event-driven communication internally to propagate state changes across simulation components.
  • Designed APIs to support deterministic simulation runs and reproducible results.

Frontend & PWA Architecture

  • Built a React-based Progressive Web App to deliver a responsive, offline-capable user experience for engineers and planners.
  • Implemented real-time visualization of simulation results, machine states, and production metrics.
  • Designed the frontend to be backend-driven, enabling rapid iteration of simulation logic without UI rewrites.

Leadership & Ownership

  • Led a cross-functional team of engineers, owning technical direction, architecture decisions, and code quality.
  • Broke down complex simulation requirements into modular components and assigned clear ownership across the team.
  • Conducted design reviews, enforced coding standards, and mentored junior engineers throughout development.
